Module 4: Bonus Modules

4 เทคนิคต่อยอดขั้นสูง สำหรับคนที่ผ่าน Module 1-3 แล้วอยากขยายระบบให้ไกลขึ้น: Automation ภายนอกด้วย n8n, งานภาพ, งานประชุม และการสร้าง Instructions ระดับโปร

Bonus 1

ใช้ Claude Cowork สร้าง n8n Workflow

เป้าหมายบทนี้: ให้ Claude ออกแบบและสร้าง workflow automation บน n8n ให้เรา แม้เราไม่เคยเขียน workflow มาก่อน

n8n คืออะไร และทำไมต้องใช้คู่กับ Claude

n8n คือเครื่องมือ automation แบบ open-source ที่ต่อแอปเป็น workflow ได้อิสระมาก (คล้าย Zapier แต่ยืดหยุ่นกว่าและคุมเองได้) จุดที่คนติดคือ "การสร้าง workflow ครั้งแรกยาก" — และนี่คือจุดที่ Claude เข้ามาช่วย: เราบอกสิ่งที่อยากได้เป็นภาษาคน แล้วให้ Claude ออกแบบและเขียนตัว workflow ให้เลย

วิธีทำ

  1. อธิบายงาน automation ที่ต้องการให้ Claude ฟังแบบละเอียด: อะไรเป็นตัวจุดชนวน (trigger) → ทำอะไรต่อทีละขั้น → ผลลัพธ์ไปที่ไหน
  2. ให้ Claude ออกแบบ workflow และสร้างเป็นไฟล์ JSON ของ n8n (n8n รองรับการ import workflow จาก JSON)
  3. นำไฟล์ไป Import ใน n8n แล้วใส่ credentials ของแอปที่เกี่ยวข้อง
  4. ทดสอบรัน ถ้าติดตรงไหน แคปหน้าจอ/คัดลอก error กลับมาให้ Claude แก้ให้
ช่วยสร้าง n8n workflow ให้ฉัน: - Trigger: มีคน submit ฟอร์มสั่งซื้อ (webhook) - ขั้นที่ 1: บันทึกข้อมูลออเดอร์ลง Google Sheets - ขั้นที่ 2: ส่งอีเมลยืนยันหาลูกค้า (มีชื่อ สินค้า ยอดเงิน) - ขั้นที่ 3: ถ้ายอดเกิน 5,000 บาท แจ้งเตือนทีมขายทาง LINE สร้างเป็นไฟล์ JSON ที่ import เข้า n8n ได้เลย พร้อมอธิบายว่าแต่ละ node ทำอะไร และต้องใส่ credentials ตรงไหนบ้าง
แบ่งงานให้ถูกตัว: งานที่ต้อง "คิด/วิเคราะห์/เขียน" ให้ Claude ทำ — งานที่ "ส่งต่อข้อมูลอัตโนมัติวันละร้อยรอบ" ให้ n8n ทำ และสองอย่างนี้ต่อกันได้: n8n เรียกใช้ Claude ผ่าน API ในขั้นตอนที่ต้องใช้สมองก็ได้
แบบฝึกหัด
เลือกงานส่งต่อข้อมูล 1 งานในธุรกิจคุณ ให้ Claude สร้าง n8n workflow JSON แล้ว import ทดสอบรันจริง 1 รอบ
Bonus 2

ใช้ Cowork เรียกใช้ Skill เจนรูป

เป้าหมายบทนี้: ตั้งระบบงานภาพ: ให้ AI ช่วยตั้งแต่คิดคอนเซปต์ เขียน prompt ไปจนถึงสร้างภาพผ่านเครื่องมือที่เชื่อมไว้

แนวทางการทำงานภาพด้วย Cowork

  1. เชื่อมเครื่องมือสร้างภาพ: ผ่าน Connector ที่รองรับ เช่น Canva (สร้างงานออกแบบจากเนื้อหาที่ Claude เตรียม) หรือเครื่องมือเจนรูปอื่นที่เชื่อมผ่าน MCP/Zapier ได้
  2. สร้าง Skill "Art Direction" ของแบรนด์: เก็บโทนสี สไตล์ภาพ สิ่งที่ห้ามทำ ไว้ใน Skill เพื่อให้งานภาพทุกชิ้นไปทางเดียวกัน
  3. ให้ Claude เขียน image prompt ให้: จุดที่คนส่วนใหญ่พลาดคือเขียน prompt ภาพไม่เก่ง — ให้ Claude แปลงโจทย์การตลาดเป็น prompt ภาพที่ละเอียดแทนเรา
ชื่อ Skill: Brand Image Prompter ใช้เมื่อ: ต้องการภาพประกอบคอนเทนต์หรือโฆษณาของแบรนด์ ขั้นตอน: 1. รับโจทย์: ภาพใช้ทำอะไร ลงช่องทางไหน สัดส่วนเท่าไร สื่ออารมณ์อะไร 2. ยึด Art Direction ของแบรนด์: โทนสีอบอุ่น แสงธรรมชาติ คนจริงไม่ใช่การ์ตูน ห้ามใส่ข้อความในภาพ 3. เขียน image prompt ภาษาอังกฤษแบบละเอียด 3 เวอร์ชัน (มุมกล้อง แสง อารมณ์ องค์ประกอบ) 4. ถ้ามีเครื่องมือสร้างภาพเชื่อมอยู่ ให้สร้างภาพจาก prompt ที่ดีที่สุด ถ้าไม่มี ให้ส่ง prompt ทั้ง 3 ให้ผู้ใช้นำไปใช้เอง
ข้อควรระวังงานภาพ AI: ตรวจภาพก่อนใช้จริงเสมอ (นิ้วมือ ตัวอักษรเพี้ยน โลโก้แปลกปลอม), อย่าเจนภาพเลียนแบบบุคคลจริงหรืองานมีลิขสิทธิ์ และเช็กเงื่อนไขการใช้เชิงพาณิชย์ของเครื่องมือที่ใช้
แบบฝึกหัด
สร้าง Skill Art Direction ของแบรนด์คุณ แล้วให้ Claude ผลิต image prompt สำหรับคอนเทนต์จริง 1 ชิ้น นำไปเจนภาพและเทียบผลกับ prompt ที่คุณเคยเขียนเอง
Bonus 3

Connect กับ AI Meeting Recorder

เป้าหมายบทนี้: ปิดวงจรงานประชุม: อัดเสียง → ถอดความ → ให้ Claude สรุปและแตกงานต่ออัตโนมัติ

วงจรงานประชุมอัตโนมัติ

  1. อัด + ถอดความ: ใช้เครื่องมือ meeting recorder ที่มี transcript เช่น เครื่องมืออัดเสียงที่ถอดความได้ หรือฟีเจอร์บันทึกของโปรแกรมประชุมที่ใช้อยู่
  2. ส่งเข้าระบบ: ตั้งให้ transcript ถูกบันทึกลงโฟลเดอร์ที่ Cowork เข้าถึงได้ (หรือเชื่อมผ่าน Connector ของเครื่องมือนั้นถ้ามี)
  3. ให้ Claude ประมวลผล: สรุปประชุม แตกรายการงาน (ใครทำอะไร เมื่อไร) และร่างอีเมล follow-up
  4. ต่อยอดอัตโนมัติ: สร้างเป็น Scheduled Task ให้กวาดไฟล์ transcript ใหม่ทุกเย็นแล้วสรุปให้เอง
ในโฟลเดอร์ "ประชุม" มีไฟล์ transcript การประชุมล่าสุด ช่วยทำ 3 อย่าง: 1. สรุปการประชุม: ประเด็นหลัก การตัดสินใจ และเหตุผล (ไม่เกิน 1 หน้า) 2. แตกรายการ Action Items เป็นตาราง: งาน / ผู้รับผิดชอบ / กำหนดเสร็จ ถ้าในประชุมไม่ได้ระบุคนหรือวัน ให้ทำเครื่องหมาย [ต้องยืนยัน] 3. ร่างอีเมลสรุปส่งผู้เข้าประชุม แนบทั้งสรุปและ Action Items บันทึกทั้งหมดเป็นไฟล์ "สรุปประชุม-[วันที่].docx"
มารยาทและกฎหมาย: แจ้งผู้เข้าประชุมทุกครั้งก่อนอัดเสียง และระวังการส่งเนื้อหาประชุมที่เป็นความลับเข้าเครื่องมือภายนอก — ตรวจนโยบายข้อมูลของบริษัทคุณก่อนตั้งระบบนี้
แบบฝึกหัด
นำ transcript ประชุมจริง 1 รายการ (หรือพิมพ์บันทึกการประชุมคร่าว ๆ) มารันด้วย Prompt ด้านบน แล้วเทียบกับสรุปที่คุณเคยทำมือว่าครบกว่ากันแค่ไหน
Bonus 4

เครื่องมือ Gen Instructions สำหรับใช้ใน Project

เป้าหมายบทนี้: ใช้ AI ช่วยเขียน Instructions ของ Project ให้เอง — เขียนครั้งเดียว ดีทุกแชต

ทำไม Instructions ถึงสำคัญที่สุดใน Project

จาก Module 1 บทที่ 7 เรารู้แล้วว่า Instructions คือสิ่งที่ทุกแชตใน Project ทำตามอัตโนมัติ — แต่คนส่วนใหญ่เขียนสั้นเกินไปหรือกว้างเกินไป เทคนิคของบทนี้คือ ให้ Claude สัมภาษณ์เราแล้วเขียน Instructions ให้เอง

เครื่องมือ: Instructions Generator

คุณคือผู้เชี่ยวชาญการเขียน Project Instructions สำหรับ Claude ฉันกำลังจะสร้าง Project ใหม่ ช่วยสัมภาษณ์ฉันทีละข้อ (ถามทีละคำถาม รอคำตอบก่อนถามต่อ): 1. Project นี้ใช้ทำงานอะไร ผลงานหลักคืออะไร 2. ใครใช้ Project นี้บ้าง และผลงานส่งให้ใครดู 3. มีกติกา/ข้อห้ามอะไรที่ต้องยึดเสมอ (โทนภาษา แบรนด์ ข้อมูลที่ห้ามแต่งเอง) 4. รูปแบบผลลัพธ์มาตรฐานที่ต้องการ 5. ตัวอย่างงานที่เคยพอใจ (ถ้ามี ให้แนบ) เมื่อได้คำตอบครบ เขียน Project Instructions ฉบับสมบูรณ์ให้ โดย: - แบ่งหมวด: บทบาท / บริบทธุรกิจ / กติกา / รูปแบบผลลัพธ์ / สิ่งที่ห้ามทำ - เขียนเป็นคำสั่งที่ชัดเจน วัดได้ ไม่กำกวม - ปิดท้ายด้วยรายการไฟล์ Knowledge ที่แนะนำให้อัปโหลดเข้า Project

เช็กลิสต์ Instructions ที่ดี

  • อ่านแล้วรู้ทันทีว่า Project นี้ทำอะไร ให้ใคร
  • มีกติกาที่ "วัดได้" เช่น "ไม่เกิน 5 บรรทัด" ไม่ใช่ "สั้น ๆ"
  • ระบุแหล่งข้อมูลที่ให้ยึด และสิ่งที่ห้ามแต่งเอง
  • มีรูปแบบผลลัพธ์มาตรฐานชัดเจน
  • ยาวพอดี — ครอบคลุมแต่ไม่ยัดทุกอย่างจนขัดกันเอง
โปรเจกต์ท้าย Module
ใช้ Instructions Generator สร้าง Instructions ให้ Project ที่คุณสร้างไว้ใน Module 1 ใหม่ทั้งหมด แล้วเทียบคุณภาพงานก่อน-หลัง ด้วยคำสั่งเดียวกัน
