Module 1: Foundations
เริ่มต้นจนใช้เป็นระบบ

11 บทเรียนพื้นฐานที่พาคุณจากคนที่ไม่เคยใช้ Claude เลย ไปจนถึงคนที่มี "ระบบ AI ทำงานแทน" ของตัวเอง — แต่ละบทมีตัวอย่าง Prompt และแบบฝึกหัดให้ลองทำตามทันที

Lesson 1

Intro to Claude — Claude คืออะไร ใช้ทำอะไรได้บ้าง

เป้าหมายบทนี้: เข้าใจว่า Claude คืออะไร ต่างจาก AI ตัวอื่นอย่างไร และเห็นภาพรวมว่าธุรกิจนำไปใช้ทำอะไรได้จริงบ้าง

Claude คืออะไร

Claude คือ AI ผู้ช่วยอัจฉริยะที่พัฒนาโดยบริษัท Anthropic จุดเด่นคือความสามารถในการเข้าใจบริบทยาว ๆ ทำงานกับไฟล์และเอกสารจริง เขียนงานคุณภาพสูง และ "ลงมือทำงานแทนเรา" ได้จริงผ่านโหมดการทำงานแบบ Agent — ไม่ใช่แค่ตอบคำถามเป็นข้อความ

Claude ใช้งานได้หลายช่องทาง: ผ่านเว็บ (claude.ai), แอปมือถือ, แอป Desktop (Windows/Mac) และ Claude Cowork ซึ่งเป็นโหมดการทำงานแบบ Agent บนแอป Desktop ที่ให้ Claude เข้าถึงโฟลเดอร์งานของเรา อ่านไฟล์ สร้างไฟล์ และทำงานหลายขั้นตอนต่อเนื่องจนจบงานได้เอง

ภาพรวมการใช้ AI ในงานจริง (Business Use Cases)

  • งานเอกสาร: สรุปรายงาน ร่างอีเมล เขียน Proposal ทำสไลด์นำเสนอ
  • งานข้อมูล: รวมไฟล์ Excel หลายไฟล์ วิเคราะห์ยอดขาย หา Insight สร้าง Dashboard
  • งานการตลาด: เขียน Content, วิเคราะห์โฆษณาคู่แข่ง, สร้าง Landing Page, วางแผนแคมเปญ
  • งานบริหาร: สรุปประชุม ติดตามงาน ทำรายงานประจำสัปดาห์อัตโนมัติ
  • งานซ้ำ ๆ: ตั้งเวลาให้ AI ทำแทนทุกวัน/ทุกสัปดาห์ โดยไม่ต้องสั่งใหม่
วิธีคิดที่ถูกต้อง: อย่ามอง Claude เป็น "เครื่องมือค้นหา" ให้มองเป็น "พนักงานใหม่ที่เก่งมากแต่ยังไม่รู้จักธุรกิจเรา" — ยิ่งเราให้บริบทงานชัดเท่าไร ผลงานยิ่งดีเท่านั้น
แบบฝึกหัดท้ายบท
เขียนรายการงานของคุณ 10 อย่างที่ทำเป็นประจำ แล้วติ๊กว่างานไหนเข้าข่าย "งานเอกสาร / งานข้อมูล / งานซ้ำ ๆ" — รายการนี้จะเป็นโจทย์ที่เราใช้ตลอดทั้งคอร์ส
Lesson 2

Getting Started — ดาวน์โหลด & ตั้งค่า Memory

เป้าหมายบทนี้: ติดตั้ง Claude ให้พร้อมใช้ ตั้งค่า Memory ให้ AI จำบริบทงานของเรา และเตรียม environment สำหรับใช้งานจริง

ขั้นตอนเริ่มใช้งาน Claude

  1. สมัครบัญชีที่ claude.ai (ใช้อีเมลหรือบัญชี Google ได้)
  2. ดาวน์โหลด แอป Claude Desktop จาก claude.ai/download แล้วติดตั้งบนเครื่อง — Cowork ทำงานผ่านแอปนี้
  3. ล็อกอินด้วยบัญชีเดียวกัน ทั้งเว็บ มือถือ และ Desktop จะซิงก์ประวัติแชตถึงกัน
  4. ตรวจสอบแพ็กเกจ: ฟีเจอร์กลุ่ม Agent อย่าง Cowork ต้องใช้แพ็กเกจแบบเสียเงิน (Pro ขึ้นไป)

ตั้งค่า Memory ให้ AI จำบริบทงานเรา

Memory คือความสามารถที่ทำให้ Claude "จำ" ข้อมูลสำคัญเกี่ยวกับเราไว้ใช้ในแชตต่อ ๆ ไป โดยไม่ต้องอธิบายซ้ำทุกครั้ง สิ่งที่ควรให้ Claude จำตั้งแต่วันแรก:

  • คุณทำธุรกิจ/อาชีพอะไร ขายอะไร ให้ใคร
  • ตำแหน่งและหน้าที่หลักของคุณ
  • โทนภาษาที่ต้องการ (ทางการ / กันเอง) และภาษาที่ให้ตอบ
  • รูปแบบผลลัพธ์ที่ชอบ เช่น "สรุปเป็นข้อ ๆ ก่อนเสมอ"
ช่วยจำข้อมูลนี้ไว้ใช้กับการทำงานร่วมกันทุกครั้ง: - ฉันทำธุรกิจขายอุปกรณ์กีฬาออนไลน์ ลูกค้าหลักคือนักวิ่งอายุ 25-45 ปี - ฉันดูแลการตลาดและยอดขายเอง มีทีม 3 คน - ตอบเป็นภาษาไทยเสมอ โทนกระชับ ตรงประเด็น - เวลาวิเคราะห์ข้อมูล ให้สรุป Insight สำคัญ 3-5 ข้อก่อน แล้วค่อยลงรายละเอียด

เตรียม environment สำหรับใช้งานจริง

  • สร้างโฟลเดอร์กลางบนเครื่อง เช่น AI-Workspace แล้วแยกโฟลเดอร์ย่อยตามงาน (ยอดขาย / การตลาด / เอกสาร)
  • ย้ายไฟล์ที่จะให้ AI ทำงานด้วยมาไว้ในโฟลเดอร์นี้ — Cowork จะขอสิทธิ์เข้าถึงเป็นโฟลเดอร์ จัดระเบียบไว้ก่อนจะทำงานง่ายมาก
  • ตั้งชื่อไฟล์ให้สื่อความหมาย เช่น ยอดขาย-มกราคม-2026.xlsx แทน Book1.xlsx
แบบฝึกหัดท้ายบท
ติดตั้งแอป Desktop, สั่งให้ Claude จำบริบทธุรกิจของคุณด้วย Prompt ตามตัวอย่าง แล้วเปิดแชตใหม่ถามว่า "ฉันทำธุรกิจอะไร" เพื่อทดสอบว่า Memory ทำงาน
Lesson 3

Basic Functions in Claude — Prompt / Chat / File

เป้าหมายบทนี้: ใช้งานพื้นฐานได้คล่อง และรู้เทคนิคการสั่งงานให้ได้ผลลัพธ์ดีตั้งแต่ครั้งแรก

การใช้งานพื้นฐาน 3 อย่าง

  • Prompt: ข้อความคำสั่งที่เราพิมพ์บอก AI — คุณภาพของ Prompt กำหนดคุณภาพของงานโดยตรง
  • Chat: การสนทนาต่อเนื่อง — Claude จำทุกอย่างในแชตเดียวกัน จึงสั่งแก้งานต่อได้เรื่อย ๆ โดยไม่ต้องเริ่มใหม่ แต่ควรเปิดแชตใหม่เมื่อเปลี่ยนเรื่อง
  • File: แนบไฟล์ให้ Claude อ่านได้โดยตรง ทั้ง PDF, Word, Excel, CSV, รูปภาพ — ลากไฟล์ลงในช่องแชตได้เลย

เทคนิคการสั่งงานให้ได้ผลลัพธ์ดี

  1. บอกบทบาท (Role): "คุณคือนักวิเคราะห์การตลาดที่เชี่ยวชาญ e-commerce"
  2. บอกบริบท (Context): ธุรกิจอะไร สถานการณ์เป็นอย่างไร ข้อมูลอะไรเกี่ยวข้อง
  3. บอกงานให้ชัด (Task): ต้องการอะไร ขอบเขตแค่ไหน
  4. บอกรูปแบบผลลัพธ์ (Format): ตาราง? ข้อ ๆ? ความยาวเท่าไร? ภาษาอะไร?
  5. ให้ตัวอย่าง (Example): ถ้ามีตัวอย่างงานที่ชอบ แนบให้ดูเลยยิ่งดี

โครงสร้าง Prompt ที่ใช้ในงานจริง

[บทบาท] คุณคือผู้ช่วยฝ่ายขายมือโปร [บริบท] บริษัทเราขายซอฟต์แวร์บัญชีให้ SME ลูกค้าเพิ่งบ่นเรื่องการตอบแชตช้า [งาน] ช่วยร่างข้อความขอโทษลูกค้า พร้อมเสนอส่วนลด 10% สำหรับการต่ออายุ [รูปแบบ] ภาษาไทย สุภาพแต่จริงใจ ไม่เกิน 5 บรรทัด มี 2 เวอร์ชันให้เลือก
เทคนิคลัด: ถ้าไม่แน่ใจว่าจะเขียน Prompt อย่างไร ให้พิมพ์ว่า "ฉันอยากได้ [สิ่งที่ต้องการ] — ก่อนเริ่มทำ ช่วยถามคำถามที่จำเป็นกับฉันก่อน" Claude จะถามจนได้ข้อมูลครบแล้วค่อยลงมือ
แบบฝึกหัดท้ายบท
เลือกงานเขียน 1 ชิ้นจากรายการงานในบทที่ 1 แล้วสั่ง Claude ด้วยโครงสร้าง 4 ส่วน (บทบาท-บริบท-งาน-รูปแบบ) เปรียบเทียบผลลัพธ์กับการสั่งสั้น ๆ บรรทัดเดียว
Lesson 4

Features Inside Cowork — ฟีเจอร์หลักที่ช่วยทำงานแทนเรา

เป้าหมายบทนี้: เข้าใจว่า Cowork ต่างจากแชตปกติอย่างไร และรู้จักฟีเจอร์หลักที่ทำให้ AI "ทำงานแทน" ได้จริง

ทำความเข้าใจ Claude Cowork

แชตปกติ: เราถาม → AI ตอบเป็นข้อความ แล้วเราต้องเอาไปทำต่อเอง
Cowork: เรามอบหมายงาน → AI วางแผน ลงมือทำทีละขั้น เปิดไฟล์ อ่านข้อมูล สร้างไฟล์ผลลัพธ์ แล้วส่งงานที่ "เสร็จแล้ว" กลับมา — เหมือนส่งงานให้เพื่อนร่วมงานคนหนึ่งไปทำ

ฟีเจอร์หลักที่ช่วยทำงานแทนเรา

  • เข้าถึงโฟลเดอร์งาน: ให้สิทธิ์ Cowork เข้าโฟลเดอร์ที่ต้องการ แล้ว AI อ่าน-สร้าง-แก้ไฟล์ในนั้นได้เอง
  • ทำงานหลายขั้นตอนต่อเนื่อง: สั่งครั้งเดียว AI แตกงานเป็นขั้นตอนแล้วทำจนจบ เช่น "อ่านไฟล์ทั้งโฟลเดอร์ → สรุป → ทำเป็นรายงาน 1 ไฟล์"
  • สร้างไฟล์ผลลัพธ์จริง: Excel, Word, PowerPoint, PDF, หน้าเว็บ HTML — ได้ไฟล์ที่เปิดใช้ต่อได้ทันที
  • ทำงานเบื้องหลัง: มอบหมายงานแล้วไปทำอย่างอื่นได้ งานเสร็จค่อยกลับมาดู
  • ขอสิทธิ์ก่อนทำสิ่งสำคัญ: Cowork จะถามยืนยันก่อนทำสิ่งที่มีผลกระทบ เช่น ลบไฟล์ — เราคุมได้ตลอด

วิธีใช้ Cowork ให้เกิด Productivity สูงสุด

  1. สั่งเป็น "ชิ้นงาน" ไม่ใช่ "คำถาม": แทนที่จะถามว่า "วิเคราะห์ยอดขายยังไงดี" ให้สั่งว่า "วิเคราะห์ไฟล์ยอดขายในโฟลเดอร์นี้ แล้วทำสรุปเป็น PowerPoint 5 หน้า"
  2. ระบุ Definition of Done: บอกให้ชัดว่างานเสร็จหน้าตาเป็นอย่างไร ไฟล์ชื่ออะไร เก็บที่ไหน
  3. ให้ตรวจงานตัวเองก่อนส่ง: เติมท้ายคำสั่งว่า "ก่อนส่งงาน ตรวจสอบตัวเลขและความครบถ้วนอีกรอบ"
ในโฟลเดอร์ "รายงานลูกค้า" มีไฟล์ feedback ลูกค้าเดือนนี้หลายไฟล์ ช่วยอ่านทั้งหมด แล้วทำสรุปเป็นไฟล์ Word ชื่อ "สรุป-feedback-เดือนนี้.docx" ประกอบด้วย: 1. ประเด็นที่ลูกค้าชมมากที่สุด 3 อันดับ 2. ประเด็นที่ถูกร้องเรียนมากที่สุด 3 อันดับ พร้อมตัวอย่างข้อความจริง 3. ข้อเสนอแนะสิ่งที่ควรแก้ก่อน 1 อย่าง ก่อนส่งงาน ตรวจสอบว่าครอบคลุมทุกไฟล์แล้ว
แบบฝึกหัดท้ายบท
เปิด Cowork ให้สิทธิ์เข้าโฟลเดอร์ AI-Workspace ที่สร้างไว้ แล้วมอบหมายงานแรก: ให้สำรวจไฟล์ในโฟลเดอร์และทำไฟล์สรุปว่ามีไฟล์อะไรบ้าง จัดหมวดหมู่อย่างไรดี
Lesson 5

Data Analysis from Multiple Files — วิเคราะห์ข้อมูลจากหลายไฟล์

เป้าหมายบทนี้: เอาไฟล์ที่กระจัดกระจายมารวมกัน ให้ AI วิเคราะห์แบบอัตโนมัติ และได้ Insight พร้อมใช้ทันที

ปัญหาที่บทนี้แก้

ข้อมูลจริงในธุรกิจไม่เคยอยู่ไฟล์เดียว — ยอดขายอยู่ Excel หนึ่งไฟล์ต่อเดือน รายชื่อลูกค้าอยู่อีกไฟล์ ค่าโฆษณาอยู่ CSV ที่ export มา การนั่งรวมเองใช้เวลาครึ่งวัน แต่ Cowork ทำได้ในไม่กี่นาที

ขั้นตอนการทำงาน

  1. รวมไฟล์ที่เกี่ยวข้องไว้โฟลเดอร์เดียว (เช่น "วิเคราะห์-Q1")
  2. บอก Cowork ว่าไฟล์แต่ละไฟล์คืออะไร — หรือให้ AI สำรวจเองก็ได้
  3. ระบุคำถามธุรกิจที่อยากรู้ ไม่ใช่แค่ "วิเคราะห์ให้หน่อย"
  4. ให้สร้างไฟล์สรุปผลลัพธ์ที่เอาไปใช้ต่อได้
ในโฟลเดอร์ "วิเคราะห์-Q1" มีไฟล์ยอดขายราย 3 เดือน (ม.ค.-มี.ค.) และไฟล์ค่าโฆษณา ช่วยวิเคราะห์และตอบคำถามเหล่านี้: 1. ยอดขายรวมและแนวโน้มแต่ละเดือน โตขึ้นหรือลดลงกี่ % 2. สินค้าไหนขายดีสุด/แย่สุด 5 อันดับ 3. ROAS (ยอดขายต่อค่าโฆษณา) ของแต่ละเดือน 4. มีอะไรผิดปกติที่ควรเช็กไหม (ยอดหาย ตัวเลขกระโดด ข้อมูลซ้ำ) สรุปเป็น Insight สำคัญ 5 ข้อ พร้อมคำแนะนำว่าควรทำอะไรต่อ

ให้ AI วิเคราะห์แบบอัตโนมัติ — เทคนิคสำคัญ

  • ให้เช็กคุณภาพข้อมูลก่อนเสมอ: สั่งให้ AI รายงานข้อมูลที่หาย ซ้ำ หรือผิดปกติก่อนวิเคราะห์ — ป้องกันสรุปผิดเพราะข้อมูลสกปรก
  • ถามต่อได้เหมือนคุยกับนักวิเคราะห์: "เจาะเฉพาะลูกค้าใหม่ให้หน่อย" "ตัดข้อมูลเดือนที่มีโปรออกแล้วดูใหม่"
  • ให้แสดงวิธีคิด: "บอกด้วยว่าตัวเลขนี้คำนวณจากคอลัมน์ไหน" เพื่อตรวจทานได้
ข้อควรระวัง: ตรวจตัวเลขสำคัญก่อนใช้ตัดสินใจเสมอ โดยเฉพาะตัวเลขที่จะเอาไปรายงานผู้บริหารหรือใช้เรื่องเงิน — ให้ AI ชี้แหล่งที่มาของทุกตัวเลขได้ ก็ตรวจได้เร็ว
แบบฝึกหัดท้ายบท
รวมไฟล์ข้อมูลจริงของคุณ 2-3 ไฟล์ (หรือไฟล์ตัวอย่าง) ไว้โฟลเดอร์เดียว แล้วสั่งวิเคราะห์ด้วยโครงสร้าง "คำถามธุรกิจ 3-4 ข้อ + ขอ Insight 5 ข้อ" ตามตัวอย่าง
Lesson 6

Create Excel from Multiple Sources — รวมหลายไฟล์เป็นไฟล์เดียว

เป้าหมายบทนี้: รวม Excel หลายไฟล์เป็นไฟล์เดียว สร้างโครงสร้างข้อมูลใหม่ที่สะอาด และลดงาน manual แบบมหาศาล

สถานการณ์จริงที่เจอบ่อย

  • ไฟล์ยอดขายแยกรายเดือน 12 ไฟล์ อยากได้ไฟล์รวมทั้งปีไฟล์เดียว
  • แต่ละสาขาส่งไฟล์มาคนละฟอร์แมต หัวตารางไม่ตรงกัน
  • ข้อมูลลูกค้าซ้ำกันหลายไฟล์ อยากได้รายชื่อสะอาด ๆ ไม่ซ้ำ

สั่งรวมไฟล์ + จัดโครงสร้างใหม่

ในโฟลเดอร์ "ยอดขายรายเดือน" มีไฟล์ Excel 12 ไฟล์ (ม.ค.-ธ.ค.) หัวตารางอาจไม่ตรงกันทุกไฟล์ ช่วยทำสิ่งนี้: 1. รวมทุกไฟล์เป็นไฟล์เดียวชื่อ "ยอดขาย-ทั้งปี.xlsx" 2. จัดหัวตารางให้เป็นมาตรฐานเดียวกัน: วันที่ / รหัสสินค้า / ชื่อสินค้า / จำนวน / ราคา / ยอดรวม / ช่องทางขาย 3. เพิ่มคอลัมน์ "เดือน" จากชื่อไฟล์ต้นทาง 4. ลบแถวซ้ำและแถวว่าง แล้วรายงานว่าลบไปกี่แถว 5. เพิ่มชีต "สรุป" แสดงยอดรวมรายเดือนและ Top 10 สินค้า ถ้าเจอไฟล์ไหนโครงสร้างแปลกจนรวมไม่ได้ ให้แจ้งก่อน อย่าเดา

หลักการสำคัญ: ออกแบบโครงสร้างปลายทางก่อน

ก่อนสั่งรวมไฟล์ ให้ตอบ 3 คำถามนี้ก่อน แล้วใส่ในคำสั่ง:

  1. คอลัมน์สุดท้ายต้องมีอะไรบ้าง? — ระบุชื่อคอลัมน์ที่ต้องการให้ชัด
  2. 1 แถวแทนอะไร? — 1 รายการขาย? 1 ลูกค้า? 1 วัน?
  3. ข้อมูลชนกันให้ยึดอะไร? — เช่น ลูกค้าซ้ำให้ยึดข้อมูลล่าสุด
ลดงาน manual มหาศาล: งานรวมไฟล์ที่เคยใช้เวลา 3-4 ชั่วโมงต่อเดือน พอเขียนคำสั่งดี ๆ เก็บไว้ 1 ครั้ง เดือนถัดไปก็ใช้คำสั่งเดิมซ้ำได้เลย — และในบทที่ 9 เราจะตั้งเวลาให้มันทำเองอัตโนมัติ
แบบฝึกหัดท้ายบท
หาไฟล์ Excel/CSV ที่เกี่ยวข้องกัน 3 ไฟล์ขึ้นไป สั่งรวมเป็นไฟล์เดียวโดยกำหนดหัวตารางปลายทางเอง พร้อมชีตสรุป แล้วเปิดไฟล์ตรวจว่าตรงตามสั่งไหม
Lesson 7

Cowork + Claude Projects — จัดระบบงานด้วย Projects

เป้าหมายบทนี้: ใช้ Projects จัดระบบงาน ทำให้ AI เข้าใจ context ระยะยาว และทำงานเป็น workflow

Projects คืออะไร

Project คือ "ห้องทำงาน" ที่รวมทุกอย่างของงานหนึ่งเรื่องไว้ด้วยกัน: ไฟล์ความรู้ (Knowledge), คำสั่งประจำ (Instructions) และแชตทั้งหมดที่เกี่ยวกับงานนั้น — ทุกแชตใน Project จะรู้บริบทเดียวกันโดยอัตโนมัติ ไม่ต้องอธิบายซ้ำ

โครงสร้าง Project ที่ดี

  • Instructions (คำสั่งประจำ Project): บอกว่า Project นี้ทำงานอะไร โทนภาษา รูปแบบผลลัพธ์ กติกาที่ต้องยึด — ทุกแชตใน Project จะทำตามอัตโนมัติ
  • Knowledge (ไฟล์ความรู้): อัปโหลดเอกสารที่ต้องใช้อ้างอิงประจำ เช่น รายละเอียดสินค้า ราคา นโยบายบริษัท Brand Guideline
  • แชตแยกตามงานย่อย: 1 แชต = 1 งาน เช่น "คอนเทนต์สัปดาห์ 29" แยกจาก "ตอบคำถามลูกค้า"
ตัวอย่าง Project Instructions — "ทีมการตลาด" Project นี้ใช้ทำงานการตลาดของร้าน [ชื่อร้าน] - สินค้าและราคาอ้างอิงจากไฟล์ price-list ใน Knowledge เท่านั้น ห้ามแต่งเอง - โทนแบรนด์: เป็นกันเอง จริงใจ ไม่ hard sell ใช้ "เรา/คุณ" - คอนเทนต์ทุกชิ้นต้องมี: hook ใน 1 บรรทัดแรก + CTA ปิดท้าย - ก่อนส่งงานเขียนทุกครั้ง เสนอ 2 เวอร์ชันให้เลือก

ใช้ AI ทำงานเป็น Workflow

พอมี Project ที่บริบทแน่นแล้ว งานจะไหลเป็นระบบ เช่น workflow คอนเทนต์รายสัปดาห์:

  1. แชต 1: "วางแผนหัวข้อคอนเทนต์สัปดาห์นี้ 5 หัวข้อจากสินค้าเด่นเดือนนี้"
  2. แชต 2: "เขียนคอนเทนต์ตามแผน หัวข้อละ 1 โพสต์"
  3. แชต 3: "สรุปผลโพสต์สัปดาห์ที่แล้วจากไฟล์ stats ที่แนบ แล้วปรับแผนสัปดาห์หน้า"
แนวคิด "AI เข้าใจ context ระยะยาว": ยิ่ง Knowledge และ Instructions ดี AI ยิ่งทำงานเหมือนพนักงานที่อยู่กับเรามานาน — ลงทุนเวลาจัด Project ดี ๆ 1 ชั่วโมง ประหยัดเวลาอธิบายงานไปอีกหลายร้อยครั้ง
แบบฝึกหัดท้ายบท
สร้าง Project แรกของคุณ 1 อัน พร้อม Instructions อย่างน้อย 5 ข้อ และอัปโหลดไฟล์ Knowledge อย่างน้อย 2 ไฟล์ แล้วทดสอบสั่งงานเดิมที่เคยสั่งนอก Project เทียบคุณภาพผลลัพธ์
Lesson 8

Browser Plugin + Skills Setup — เชื่อม AI กับการใช้งานจริงบนเว็บ

เป้าหมายบทนี้: ติดตั้ง Plugin บน Chrome, เริ่มใช้ Skills และเชื่อม AI เข้ากับงานที่ทำบนเว็บทุกวัน

ติดตั้ง Plugin บน Chrome

  1. ติดตั้งส่วนขยาย Claude for Chrome จาก Chrome Web Store (ค้นชื่อ "Claude" ผู้พัฒนา Anthropic)
  2. ล็อกอินด้วยบัญชี Claude เดียวกับที่ใช้บนแอป
  3. ปักหมุดไอคอนไว้บนแถบเครื่องมือ เพื่อเรียกใช้ได้จากทุกหน้าเว็บ

สิ่งที่ทำได้เมื่อมี Claude บนเบราว์เซอร์: สรุปหน้าเว็บ/บทความยาว ๆ ที่เปิดอยู่, ถาม-ตอบจากเนื้อหาในหน้านั้น, ช่วยร่างข้อความในฟอร์มหรืออีเมลที่กำลังเขียน และมอบหมายให้ทำงานบนเว็บแทนเราได้ในงานที่รองรับ

ความปลอดภัย: ให้สิทธิ์ AI ทำงานบนเว็บเฉพาะเว็บไซต์ที่ไว้ใจ และตรวจก่อนยืนยันทุกครั้งเมื่อ AI จะกดส่งฟอร์ม สั่งซื้อ หรือทำอะไรที่ย้อนกลับไม่ได้

เริ่มใช้ Skills

Skill คือ "คู่มือทำงานสำเร็จรูป" ที่สอนให้ Claude ทำงานเฉพาะทางแบบเดียวกันทุกครั้ง เช่น Skill สร้างไฟล์ Excel, Skill ทำสไลด์, Skill ตามสูตรเฉพาะของเราเอง — เปิดใช้จากหน้า Settings > Capabilities/Skills ได้ทั้ง Skill สำเร็จรูปและ Skill ที่เราสร้างเอง (Module 3 จะพาสร้างละเอียด)

  • Skill สำเร็จรูปที่ควรเปิดไว้: สร้างเอกสาร Word / Excel / PowerPoint / PDF
  • Skill ทำงานอัตโนมัติเมื่อ Claude เห็นว่างานตรงกับ Skill นั้น — หรือเรียกใช้ตรง ๆ ก็ได้
แบบฝึกหัดท้ายบท
ติดตั้ง Claude บน Chrome แล้วลองใช้สรุปบทความยาว 1 บทความที่เปิดอยู่ จากนั้นเปิดใช้ Skills สร้างเอกสารในแอป และสั่งงานที่ให้ผลลัพธ์เป็นไฟล์ PowerPoint 1 งาน
Lesson 9

Creating Scheduled Tasks — ตั้ง AI ให้ทำงานอัตโนมัติ

เป้าหมายบทนี้: เปลี่ยนงานซ้ำ ๆ ให้ AI ทำแทนตามตารางเวลา และใช้ในงานธุรกิจจริงอย่างรายงานและ tracking

แนวคิด: งานซ้ำ → ให้ AI ทำแทน

งานที่ "ทำเหมือนเดิมทุกครั้ง แค่ข้อมูลเปลี่ยน" คือเป้าหมายอันดับหนึ่งของ Automation เช่น รายงานยอดขายประจำสัปดาห์ สรุปข่าวในอุตสาหกรรมทุกเช้า เช็กสถานะโปรเจกต์ทุกวันศุกร์ — Scheduled Tasks ให้เราตั้งคำสั่ง 1 ครั้ง แล้ว Claude ทำเองตามรอบเวลาโดยไม่ต้องสั่งใหม่

วิธีสร้าง Scheduled Task ที่ดี

  1. เริ่มจากคำสั่งที่ทำมือแล้วเวิร์ก: ทดลองสั่งงานแบบปกติจนได้ผลลัพธ์ที่พอใจก่อน แล้วค่อยแปลงเป็นงานตั้งเวลา
  2. เขียนคำสั่งให้ครบจบในตัว: ตอนงานรันอัตโนมัติเราไม่ได้อยู่ตอบคำถาม — ระบุแหล่งข้อมูล รูปแบบผลลัพธ์ และที่เก็บไฟล์ให้ชัด
  3. กำหนดรอบเวลา: ทุกเช้า / ทุกวันจันทร์ 8:00 / ทุกสิ้นเดือน ตามจังหวะงานจริง
  4. บอกวิธีแจ้งผล: ให้สรุปสั้น ๆ ว่าทำอะไรไป และถ้าติดปัญหาให้บันทึกไว้ว่าติดอะไร
[Scheduled Task — ทุกวันจันทร์ 08:00] สรุปรายงานยอดขายสัปดาห์ที่ผ่านมา: 1. อ่านไฟล์ยอดขายล่าสุดในโฟลเดอร์ "ยอดขายรายวัน" 2. เทียบกับสัปดาห์ก่อนหน้า: ยอดรวม, จำนวนออเดอร์, สินค้า Top 5 3. สร้างไฟล์ "รายงานสัปดาห์-[วันที่].docx" ในโฟลเดอร์ "รายงาน" 4. ถ้าไฟล์ข้อมูลหายหรือผิดปกติ ให้บันทึกปัญหาไว้ในรายงานแทนการเดาตัวเลข

ตัวอย่างใช้ในธุรกิจจริง

  • รายงาน: รายงานยอดขาย/การตลาดรายสัปดาห์ส่งผู้บริหาร
  • Tracking: ติดตามสถานะงานทีม สรุปสิ่งที่เสร็จ/ค้าง/เลยกำหนดทุกเย็นวันศุกร์
  • Monitoring: สรุปข่าวและความเคลื่อนไหวของตลาด/คู่แข่งทุกเช้า
กฎทอง: สัปดาห์แรกให้ตรวจผลลัพธ์ของงานอัตโนมัติทุกครั้ง พอมั่นใจแล้วค่อยปล่อยยาว และกลับมาสุ่มตรวจเป็นระยะ
แบบฝึกหัดท้ายบท
เลือกงานซ้ำ 1 งานจากรายการในบทที่ 1 เขียนคำสั่งแบบ "ครบจบในตัว" ทดลองรันมือ 1 ครั้ง แล้วตั้งเป็น Scheduled Task รอบแรกของคุณ
Lesson 10

Creating Plugins — สร้าง Plugin ใช้งานเอง

เป้าหมายบทนี้: เข้าใจว่า Plugin คืออะไร สร้างใช้เองได้ ต่อกับ API/ระบบอื่นได้ และขยายความสามารถ AI แบบ custom

Plugin คืออะไร ต่างจาก Skill อย่างไร

SkillPlugin / Connector
คืออะไรคู่มือ + ขั้นตอนการทำงาน สอนให้ AI ทำงานแบบที่เรากำหนดส่วนขยายที่เพิ่ม "ความสามารถใหม่" เช่น ต่อกับระบบภายนอก มีเครื่องมือใหม่ให้ AI ใช้
เหมาะกับงานที่ทำด้วยความรู้ + ไฟล์ เช่น สูตรวิเคราะห์ รูปแบบรายงานงานที่ต้องคุยกับระบบอื่น เช่น ดึงข้อมูลจาก API ภายใน ส่งข้อความเข้าระบบแชตทีม
ความยากในการสร้างง่าย — เขียนเป็นภาษาคนได้เลยปานกลาง — แต่ให้ Claude ช่วยสร้างได้

แนวทางสร้าง Plugin ของตัวเอง

  1. เริ่มจากความสามารถที่อยากได้: เช่น "อยากให้ AI ดึงสถานะออเดอร์จากระบบหลังบ้านของร้านได้"
  2. เช็กว่ามี Connector สำเร็จรูปไหมก่อน: ระบบดัง ๆ ส่วนใหญ่มีให้เชื่อมได้เลย (Module 2 จะลงลึก) — สร้างเองเฉพาะเมื่อไม่มี
  3. ให้ Claude ช่วยสร้าง: อธิบายว่าระบบเรามี API อะไร อยากให้เรียกใช้แบบไหน แล้วให้ Claude เขียนตัวเชื่อม (เช่น MCP server ขนาดเล็ก) ให้ พร้อมสอนวิธีติดตั้ง
  4. ทดสอบกับข้อมูลทดลองก่อนใช้กับของจริง
ฉันมีระบบหลังบ้านที่มี API (มีเอกสาร API แนบมาให้) อยากให้ Claude เรียกดู "สถานะออเดอร์" และ "สต๊อกสินค้า" ได้จากในแชต ช่วยสร้างตัวเชื่อมต่อ (MCP server) ให้หน่อย โดย: - มีเครื่องมือ 2 ตัว: ดูสถานะออเดอร์จากเลขออเดอร์ / เช็กสต๊อกจากรหัสสินค้า - อ่านข้อมูลอย่างเดียว ห้ามมีคำสั่งแก้ไขหรือลบ - สอนวิธีติดตั้งและตั้งค่าใน Claude Desktop แบบ step-by-step
หลักปลอดภัยเมื่อขยายความสามารถ AI: เริ่มจากสิทธิ์ "อ่านอย่างเดียว" เสมอ, อย่าฝัง API key ไว้ในที่ที่คนอื่นเห็น และให้สิทธิ์เท่าที่งานต้องใช้เท่านั้น
แบบฝึกหัดท้ายบท
เขียน "สเปก Plugin ในฝัน" ของคุณ 1 ตัว: เชื่อมกับระบบอะไร มีเครื่องมือกี่ตัว แต่ละตัวรับอะไร-คืนอะไร แล้วลองให้ Claude ประเมินว่าสร้างได้เลยไหมหรือมี Connector สำเร็จรูปอยู่แล้ว
Lesson 11

Putting It All Together — รวมทุกอย่างเป็น "ระบบ AI ทำงานแทน"

เป้าหมายบทนี้: ประกอบทุกอย่างจาก 10 บทที่ผ่านมาเป็น workflow จริงแบบ end-to-end สำหรับธุรกิจของคุณ

สูตรประกอบระบบ

  1. Memory + Project = AI ที่รู้จักธุรกิจเรา (บริบท)
  2. Cowork + ไฟล์ = AI ที่ลงมือทำงานจริงได้ (แรงงาน)
  3. Skills = มาตรฐานงานที่ทำเหมือนกันทุกครั้ง (คุณภาพ)
  4. Connectors/Plugins = ต่อกับเครื่องมือที่ใช้อยู่ (การเชื่อมโยง)
  5. Scheduled Tasks = ทำงานเองโดยไม่ต้องสั่ง (อัตโนมัติ)

Use Case ธุรกิจแบบ End-to-End: ร้านค้าออนไลน์

ตัวอย่างการประกอบระบบจริงของร้านค้าออนไลน์ 1 ร้าน:

  • จันทร์ 08:00 (Scheduled): AI รวมไฟล์ยอดขายสัปดาห์ก่อนจากทุกช่องทาง → สร้าง Excel รวม + รายงานสรุปผู้บริหาร
  • จันทร์ 09:00 (Project การตลาด): สั่งวางแผนคอนเทนต์สัปดาห์นี้ โดยอิงสินค้าขายดีจากรายงานเมื่อเช้า
  • ระหว่างสัปดาห์ (Cowork): งาน ad-hoc เช่น วิเคราะห์ feedback ลูกค้า ร่างข้อเสนอ ตอบเอกสาร
  • ศุกร์ 17:00 (Scheduled): AI สรุปงานที่เสร็จ/ค้างของสัปดาห์ + เตรียมวาระประชุมจันทร์หน้า

ลงมือออกแบบระบบของคุณเอง

ฉันจะออกแบบ "ระบบ AI ทำงานแทน" สำหรับธุรกิจของฉัน นี่คือรายการงานประจำของฉัน: [วางรายการงาน 10 อย่างจากบทที่ 1] ช่วยวิเคราะห์และเสนอ: 1. งานไหนควรเป็น Scheduled Task (ระบุรอบเวลาที่เหมาะ) 2. งานไหนควรทำผ่าน Project (เสนอชื่อ Project และ Instructions คร่าว ๆ) 3. งานไหนควรสร้างเป็น Skill เพราะมีสูตรตายตัว 4. งานไหนต้องใช้ Connector เชื่อมระบบอื่น 5. เรียงลำดับว่าควรลงมือสร้างอะไรก่อน-หลัง ใน 30 วันแรก
