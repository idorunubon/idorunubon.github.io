Module 3: Skills
สร้างเครื่องมือธุรกิจ 5 ตัว

Skill คือการเปลี่ยน "วิธีทำงานที่เวิร์ก" ให้กลายเป็นเครื่องมือถาวรที่เรียกใช้ได้ทุกเมื่อ — Module นี้เริ่มจากวิธีติดตั้ง แล้วพาสร้าง 5 Skills ธุรกิจแบบจับมือทำ พร้อมโครง Skill ให้นำไปปรับใช้ได้เลย

Setup

วิธีติดตั้งและสร้าง Skills

เป้าหมายบทนี้: เข้าใจโครงสร้างของ Skill และติดตั้ง/สร้าง Skill แรกได้

Skill ประกอบด้วยอะไร

Skill คือชุดคำสั่ง (เขียนเป็นภาษาคนธรรมดา) ที่บอก Claude ว่า งานนี้ทำเมื่อไร ทำอย่างไรทีละขั้น และผลลัพธ์หน้าตาแบบไหน โครงสร้างหลักมี 3 ส่วน:

  1. ชื่อ + คำอธิบาย: ใช้เมื่อไร — Claude ใช้ส่วนนี้ตัดสินใจหยิบ Skill มาใช้อัตโนมัติ
  2. ขั้นตอนการทำงาน: step-by-step ที่อยากให้ทำเหมือนกันทุกครั้ง
  3. รูปแบบผลลัพธ์: โครงรายงาน/ไฟล์ที่ต้องส่งมอบ

วิธีติดตั้ง

  1. ไปที่ Settings → Capabilities/Skills ใน Claude — เปิดใช้ Skills สำเร็จรูป หรือเพิ่ม Skill ใหม่ของเราเอง
  2. วิธีที่ง่ายที่สุด: ให้ Claude สร้าง Skill ให้ — บอกว่าอยากได้ Skill ทำอะไร แล้วให้ Claude เขียนไฟล์ Skill ตามมาตรฐานให้เลย
  3. ทดสอบโดยสั่งงานที่ตรงกับ Skill แล้วดูว่า Claude หยิบ Skill ไปใช้และได้ผลลัพธ์ตามรูปแบบที่วางไว้
ช่วยสร้าง Skill ให้ฉันตามสเปกนี้: - ชื่อ: [ชื่อ Skill] - ใช้เมื่อ: [สถานการณ์ที่ควรหยิบ Skill นี้มาใช้] - ขั้นตอน: [ขั้นตอนการทำงานที่อยากให้ทำทุกครั้ง] - ผลลัพธ์: [โครงรายงาน/ไฟล์ที่ต้องได้] เขียนเป็นไฟล์ Skill ตามมาตรฐานของ Claude ให้เลย พร้อมบอกวิธีติดตั้ง
5 Skills ต่อจากนี้ ใช้สูตรเดียวกันหมด: เรากำหนด "ใช้เมื่อไร-ขั้นตอน-ผลลัพธ์" แล้วให้ Claude ประกอบเป็น Skill — คุณสามารถคัดลอกโครงในแต่ละบทไปสร้างตามได้ทันที
Skill #1

AdSpy — วิเคราะห์โฆษณาคู่แข่ง

ใช้ทำอะไร: วิเคราะห์ Ads คู่แข่ง หา winning ads แล้วนำ pattern ที่ได้มาต่อยอดยิงแอดของเราเอง

แหล่งข้อมูลที่ใช้

ใช้ข้อมูลโฆษณาที่เปิดเผยสาธารณะ เช่น Facebook/Meta Ad Library (คลังโฆษณาที่ Meta เปิดให้ทุกคนดูได้), TikTok Creative Center รวมถึงภาพ/แคปหน้าจอโฆษณาคู่แข่งที่เราเก็บมา — แนบให้ Claude วิเคราะห์ได้โดยตรง

โครง Skill: AdSpy

ชื่อ Skill: AdSpy ใช้เมื่อ: ผู้ใช้ต้องการวิเคราะห์โฆษณาคู่แข่งหรือหาไอเดียยิงแอด ขั้นตอน: 1. รับข้อมูลโฆษณาคู่แข่ง (ลิงก์ Ad Library / ภาพแคป / ข้อความโฆษณา อย่างน้อย 5 ชิ้น) 2. วิเคราะห์แต่ละชิ้น: Hook ที่ใช้, Pain point ที่แตะ, ข้อเสนอ (Offer), CTA, รูปแบบครีเอทีฟ 3. หา "winning pattern": ชิ้นไหนน่าจะเวิร์กเพราะอะไร (ดูจากระยะเวลาที่รันต่อเนื่อง จำนวนเวอร์ชัน ความถี่ที่เจอซ้ำ) 4. สรุปเป็นตาราง: คู่แข่ง / มุมที่เล่น / จุดแข็ง / ช่องว่างที่เขายังไม่เล่น ผลลัพธ์: - ตารางวิเคราะห์โฆษณาคู่แข่ง - Winning pattern 3-5 ข้อ - ร่างโฆษณาของเรา 3 เวอร์ชันที่ใช้ pattern ที่เจอ แต่ปรับเป็นจุดขายของเราเอง (ห้ามลอกข้อความคู่แข่ง)
จริยธรรมสำคัญ: เราวิเคราะห์เพื่อเรียนรู้ "โครงสร้างที่เวิร์ก" ไม่ใช่ก๊อบปี้ข้อความหรือภาพของคู่แข่ง — Skill นี้จึงบังคับให้ผลลัพธ์สุดท้ายเป็นงานต้นฉบับของเราเสมอ
แบบฝึกหัด
เก็บโฆษณาคู่แข่ง 5 ชิ้นจาก Meta Ad Library สร้าง Skill AdSpy ตามโครงด้านบน แล้วรันวิเคราะห์รอบแรก
Skill #2

Landing Page Builder — สร้างหน้าเว็บขาย

ใช้ทำอะไร: สร้างหน้าเว็บขายพร้อม copy และ structure อัตโนมัติ พร้อมใช้งานทันที

โครงสร้างหน้าขายที่พิสูจน์แล้ว

  1. Hero: พาดหัวที่บอกผลลัพธ์ที่ลูกค้าได้ + ปุ่ม CTA
  2. Pain: ปัญหาที่ลูกค้าเจอ (ให้เขารู้สึกว่า "นี่มันเรื่องของฉัน")
  3. Solution: สินค้า/บริการของเราแก้อย่างไร
  4. Proof: รีวิว ตัวเลข ผลงานจริง
  5. Offer: ราคา สิ่งที่ได้ โบนัส การรับประกัน
  6. FAQ + CTA ปิดท้าย

โครง Skill: Landing Page Builder

ชื่อ Skill: Landing Page Builder ใช้เมื่อ: ผู้ใช้ต้องการหน้าเว็บขายสินค้า/บริการ/อีเวนต์ ขั้นตอน: 1. ถามข้อมูลให้ครบก่อนเริ่ม: สินค้าคืออะไร ราคา กลุ่มเป้าหมาย จุดขาย 3 ข้อ รีวิว/หลักฐานที่มี และ CTA ที่ต้องการ (ทัก LINE / กรอกฟอร์ม / กดซื้อ) 2. เขียน copy ตามโครง 6 ส่วน (Hero → Pain → Solution → Proof → Offer → FAQ) ภาษาไทย อ่านง่าย เน้นผลลัพธ์ที่ลูกค้าได้ 3. สร้างไฟล์ HTML หน้าเดียวจบ: สวยงาม รองรับมือถือ โหลดเร็ว ใส่ copy ทั้งหมดลงไปจริง ไม่ใช่ placeholder 4. ตรวจงานก่อนส่ง: ลิงก์ CTA ทำงาน, แสดงผลบนจอมือถือได้ดี ผลลัพธ์: - ไฟล์ landing-page.html พร้อมเปิดใช้/นำขึ้นโฮสต์ทันที - สรุป copy แยกเป็นไฟล์ข้อความ เผื่อนำไปใช้ที่อื่น
พร้อมใช้งานทันที: ไฟล์ HTML ที่ได้เปิดดูในเครื่องได้เลย และนำขึ้นออนไลน์ฟรีได้หลายทาง เช่น GitHub Pages, Netlify, Vercel — หรือให้ Claude ช่วยพาอัปโหลดทีละขั้นก็ได้
แบบฝึกหัด
สร้างหน้าขายของสินค้า/บริการจริงของคุณ 1 หน้า เปิดบนมือถือดูจริง แล้วสั่งแก้อย่างน้อย 2 รอบ (เปลี่ยนพาดหัว / สลับลำดับ Proof) เพื่อฝึกวน iteration
Skill #3

Competitor Snapshot — วิเคราะห์คู่แข่งแบบรวดเร็ว

ใช้ทำอะไร: วิเคราะห์คู่แข่งแบบรวดเร็ว เข้าใจ positioning และใช้วางกลยุทธ์ธุรกิจ

ต่างจาก AdSpy อย่างไร

AdSpy เจาะเฉพาะ "โฆษณา" แต่ Competitor Snapshot มองภาพใหญ่ทั้งบริษัท: ขายอะไร ราคาเท่าไร จุดยืนในตลาดคืออะไร แข็งตรงไหน อ่อนตรงไหน — เหมาะกับตอนวางแผนกลยุทธ์รายไตรมาส หรือก่อนออกสินค้าใหม่

โครง Skill: Competitor Snapshot

ชื่อ Skill: Competitor Snapshot ใช้เมื่อ: ผู้ใช้ต้องการภาพรวมคู่แข่งเพื่อวางกลยุทธ์ ขั้นตอน: 1. รับรายชื่อคู่แข่ง 3-5 ราย + ข้อมูลที่มี (เว็บไซต์ เพจ ราคา แคตตาล็อก) ถ้าเชื่อมเครื่องมือค้นเว็บไว้ ให้ค้นข้อมูลสาธารณะเพิ่มเติม 2. วิเคราะห์รายคู่แข่ง: สินค้า/บริการหลัก, ช่วงราคา, กลุ่มลูกค้า, ข้อความหลักที่ใช้สื่อสาร (positioning), จุดแข็ง, จุดอ่อนที่สังเกตได้ 3. ทำแผนที่ positioning: แกนราคา (ถูก-แพง) x แกนความเฉพาะทาง (ทั่วไป-เฉพาะกลุ่ม) ระบุว่าใครอยู่ตรงไหน และ "ช่องว่าง" ในตลาดอยู่ตรงไหน 4. เสนอกลยุทธ์: เราควรยืนตรงไหน สื่อสารด้วยข้อความหลักว่าอะไร ผลลัพธ์: - ตารางเทียบคู่แข่งรายตัว - แผนภาพ positioning (สร้างเป็นภาพ/HTML ให้เห็นจริง) - ข้อเสนอกลยุทธ์ 3 ข้อ พร้อมเหตุผล - ข้อควรระวัง: ระบุด้วยว่าข้อมูลส่วนไหนเป็นการประเมิน ไม่ใช่ข้อเท็จจริงยืนยัน
แบบฝึกหัด
รัน Competitor Snapshot กับคู่แข่งจริง 3 ราย แล้วตอบให้ได้ 1 ประโยค: "เราต่างจากคู่แข่งเพราะ ______" — ถ้าตอบไม่ได้ แปลว่า positioning ยังไม่ชัด ให้วิเคราะห์ต่อ
Skill #4

CEO Dashboard — เห็นภาพรวมธุรกิจแบบ Real-time

ใช้ทำอะไร: สร้าง Dashboard สำหรับผู้บริหาร ดูภาพรวมธุรกิจ และใช้ตัดสินใจเร็วขึ้น

หลักการ Dashboard ที่ดี

  • ตัวเลขน้อยแต่ใช่: 5-8 ตัวเลขที่ผู้บริหารต้องเห็นทุกสัปดาห์ ไม่ใช่ทุกตัวเลขที่มี
  • เทียบกับอดีตเสมอ: ทุกตัวเลขต้องมี "เทียบกับช่วงก่อน" ถึงจะตัดสินใจได้
  • ชี้สิ่งผิดปกติ: Dashboard ที่ดีบอกเองว่า "ตรงนี้ต้องดู"

โครง Skill: CEO Dashboard

ชื่อ Skill: CEO Dashboard ใช้เมื่อ: ผู้ใช้ต้องการภาพรวมธุรกิจจากไฟล์ข้อมูลล่าสุด ขั้นตอน: 1. อ่านไฟล์ข้อมูลในโฟลเดอร์ที่กำหนด (ยอดขาย, ค่าใช้จ่าย, การตลาด, ลูกค้า) 2. คำนวณตัวชี้วัดหลัก: ยอดขายรวม, กำไรขั้นต้น, จำนวนออเดอร์, ยอดเฉลี่ยต่อออเดอร์, ลูกค้าใหม่ vs ลูกค้าเก่า, ROAS ทุกตัวเทียบกับช่วงก่อนหน้า (% เปลี่ยนแปลง) 3. สร้างไฟล์ dashboard.html: การ์ดตัวเลขหลักด้านบน, กราฟแนวโน้ม, ตาราง Top สินค้า/ช่องทาง, ดูได้ดีทั้งจอคอมและมือถือ 4. เพิ่มส่วน "สิ่งที่ควรดู" : 3 ประเด็นผิดปกติหรือโอกาสที่เห็นจากข้อมูล 5. ระบุวันที่ข้อมูลล่าสุดบน Dashboard เสมอ ผลลัพธ์: ไฟล์ dashboard.html ที่เปิดดูได้ทันที + สรุปผู้บริหาร 5 บรรทัด
ทำให้เป็น "Real-time": จับ Skill นี้คู่กับ Scheduled Task จากบทที่ 9 — ตั้งให้สร้าง Dashboard ใหม่ทุกเช้าจากไฟล์ล่าสุด เท่านี้ผู้บริหารก็เปิดไฟล์เดียวเห็นภาพธุรกิจล่าสุดทุกวัน
แบบฝึกหัด
สร้าง CEO Dashboard จากข้อมูลจริงของคุณ (หรือข้อมูลตัวอย่าง) แล้วให้คนอื่นดู 30 วินาที ถามว่าเขาเห็นอะไร — ถ้าตอบตรงประเด็นสำคัญ แปลว่า Dashboard ทำงาน
Skill #5

ICP Profiler — หา Customer ที่ใช่

ใช้ทำอะไร: หา customer ที่ใช่ วิเคราะห์กลุ่มเป้าหมาย และใช้ยิงแอดได้แม่นขึ้น

ICP คืออะไร

ICP (Ideal Customer Profile) คือโปรไฟล์ของ "ลูกค้าที่ใช่ที่สุด" — คนที่ซื้อง่าย จ่ายคุ้ม อยู่นาน และชวนเพื่อนมาซื้อต่อ การรู้ ICP ชัดทำให้ทั้งโฆษณา คอนเทนต์ และสินค้าใหม่แม่นขึ้นทั้งหมด เพราะเรารู้ว่ากำลังคุยกับใคร

โครง Skill: ICP Profiler

ชื่อ Skill: ICP Profiler ใช้เมื่อ: ผู้ใช้ต้องการวิเคราะห์กลุ่มลูกค้าเพื่อหาลูกค้าในอุดมคติ ขั้นตอน: 1. รับข้อมูลลูกค้าจริง: ไฟล์รายชื่อ/ประวัติซื้อ, รีวิว, ข้อความแชตที่ลูกค้าถามบ่อย (ให้ลบ/ปกปิดข้อมูลส่วนตัวที่ระบุตัวบุคคลก่อนวิเคราะห์) 2. แบ่งกลุ่มลูกค้า (Segment) จากพฤติกรรม: ซื้อบ่อย/ซื้อครั้งเดียว, ยอดสูง/ยอดต่ำ, ซื้อสินค้าประเภทไหน 3. หากลุ่มที่ "ใช่ที่สุด": กลุ่มไหนสร้างรายได้สูงสุดเทียบกับต้นทุนการได้มา 4. สร้าง ICP Profile ของกลุ่มนั้น: ช่วงวัย อาชีพ ปัญหาที่เขาอยากแก้ สิ่งที่ทำให้ตัดสินใจซื้อ ข้อโต้แย้งก่อนซื้อ ช่องทางที่เจอเขาได้ 5. แปลงเป็นการใช้งานจริง: แนวทาง targeting โฆษณา, ข้อความโฆษณาที่พูดกับกลุ่มนี้ตรง ๆ 3 แบบ, คอนเทนต์ที่กลุ่มนี้อยากอ่าน 5 หัวข้อ ผลลัพธ์: เอกสาร ICP Profile 1 หน้า + แผน targeting + ร่างข้อความโฆษณา 3 แบบ
ความเป็นส่วนตัวของลูกค้า: ก่อนส่งข้อมูลลูกค้าให้ AI วิเคราะห์ ให้ตัดข้อมูลระบุตัวตน (ชื่อเต็ม เบอร์ อีเมล ที่อยู่) ออกก่อน — การวิเคราะห์ ICP ใช้แค่พฤติกรรมและคุณลักษณะกลุ่ม ไม่จำเป็นต้องรู้ว่าใครเป็นใคร
